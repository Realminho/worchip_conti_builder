# 찬양 콘티 자동 추천기 + 유튜브 기반 DB 수집기

이 프로그램은 **YouTube Data API로 찬양 영상 후보를 자동 수집**하고, **NVIDIA API로 곡명/느림·빠름/분위기/여러 곡 포함 여부를 보조 분석**한 뒤, 사람이 실제 예배에 필요한 값만 검수해서 콘티를 추천하는 Streamlit 웹앱입니다.

## 핵심 구조

1. **유튜브 기반 자동 DB 입력**
   - 어노이팅, 마커스워십, 피아워십, 나비워십, 브리지임팩트, 청춘찬양단, 아가파오 워십 등 기본 찬양팀 목록 제공
   - 찬양팀 우선순위 없이, 선택한 모든 찬양팀/검색어를 동일하게 수집
   - 사용자가 직접 찬양팀과 검색어를 추가/수정 가능
   - 검색어당 수집 개수 제한 없이 YouTube `nextPageToken`을 따라 가능한 페이지를 끝까지 수집
   - YouTube 검색으로 제목, 채널, 유튜브 URL, 업로드일, 영상 길이 같은 링크/메타데이터만 저장
   - 예배실황/메들리처럼 영상 하나에 여러 곡이 들어 있어도 DB 행은 1개로 유지하고, `메들리 포함 곡 수`와 `포함 곡 목록`만 따로 저장

2. **NVIDIA AI 보조**
   - 기본 Base URL: `https://integrate.api.nvidia.com/v1`
   - 기본 모델: `minimaxai/minimax-m3`
   - YouTube 전체 수집이 끝난 뒤, 미분석 항목만 일괄 분석
   - 곡명, 빠르기, 주제, 메들리 여부, 메들리 포함 곡 수 추정
   - 설명란 또는 제목의 `곡A+곡B+곡C` 형태를 참고해서 포함 곡 목록 저장
   - 콘티 추천 후 구간별 흐름 설명 생성 가능

3. **검수형 DB**
   - 빠르기, 키, BPM, 첫 코드, 마지막 코드, 주제는 사람이 확인 후 저장
   - 악보/코드 악보는 필수 아님
   - 검수 완료된 곡만 콘티 추천에 사용

4. **콘티 추천**
   - 예: 앞부분 느린곡 3곡 이어서 / 중간 빠른곡 3곡 이어서 / 마지막 느린곡 1곡 끊어서
   - 구간별 원하는 분위기와 주제를 따로 설정 가능
   - 참고할 찬양팀 선택 가능
   - 이어부르기 구간은 키, BPM, 첫 코드, 마지막 코드, 주제 유사도를 점수화
   - 결과는 여러 콘티 세트로 추천하며, 사용자가 1안/2안/3안 중 선택 가능
   - 각 세트는 순서, 곡명, 찬양팀, 키, BPM, 주제, 유튜브 링크 중심으로 출력
   - 메들리는 1개 DB 항목이어도 콘티에서는 포함 곡 수만큼 카운트
   - 예: 빠른 메들리 5곡짜리가 있고 빠른곡 3곡 구간이면, 해당 메들리 중 3곡만 사용하는 추천도 가능


## 저장되는 정보

이 앱은 유튜브 영상 파일을 다운로드하거나 서버/DB에 저장하지 않습니다. DB에는 아래 정보만 저장합니다.

- `video_url`: 유튜브 영상 링크
- `source_video_id`: 원본 유튜브 영상 ID
- `clean_title`, `team`, `channel_title`, `published_at`, `duration_sec` 등 텍스트 메타데이터

메들리/예배실황처럼 한 영상에 여러 곡이 있을 때도 영상 파일을 복사하지 않습니다. DB에는 같은 원본 링크를 가진 메들리 항목 1개를 저장하고, 포함 곡 수와 포함 곡 목록만 저장합니다. 단독곡 영상과 메들리 안에 포함된 곡이 겹쳐도 서로 다른 항목으로 그대로 저장합니다.

## API 키 설정: secrets.toml 방식

이 앱은 Streamlit 웹앱으로 쓰는 것을 기준으로 만들었습니다. API 키는 코드에 직접 넣지 말고 `secrets.toml` 또는 Streamlit Cloud Secrets에 넣으세요.

### 로컬 실행

1. 프로젝트 폴더 안에 `.streamlit` 폴더를 만듭니다.
2. `.streamlit/secrets.toml.example` 파일을 참고해서 `.streamlit/secrets.toml` 파일을 만듭니다.
3. 아래처럼 입력합니다.

```toml
YOUTUBE_API_KEY = "YOUR_YOUTUBE_DATA_API_KEY"
NVIDIA_API_KEY = "YOUR_NVIDIA_API_KEY"
NVIDIA_BASE_URL = "https://integrate.api.nvidia.com/v1"
NVIDIA_MODEL = "minimaxai/minimax-m3"
```

또는 섹션 형태도 가능합니다.

```toml
[youtube]
api_key = "YOUR_YOUTUBE_DATA_API_KEY"

[nvidia]
api_key = "YOUR_NVIDIA_API_KEY"
base_url = "https://integrate.api.nvidia.com/v1"
model = "minimaxai/minimax-m3"
```

중요: `.streamlit/secrets.toml` 파일은 절대 깃허브에 올리지 마세요. `.gitignore`에 이미 제외해두었습니다.

### Streamlit Cloud 배포

1. GitHub 저장소에 이 프로젝트를 올립니다.
2. Streamlit Cloud에서 새 앱을 생성합니다.
3. Main file path를 `app.py`로 지정합니다.
4. App settings > Secrets에 로컬 `secrets.toml` 내용을 그대로 붙여넣습니다.
5. Deploy 합니다.

## 사용 순서

1. 왼쪽 메뉴에서 **1. DB 자동 수집** 선택
2. 기본값은 모든 찬양팀 선택입니다. 실제 영상 파일은 다운로드하지 않고, 필요한 경우만 선택을 줄이거나 **자동 수집 찬양팀 직접 추가/수정**에서 새 팀 추가
3. 수집은 검색어당 50개씩 페이지를 넘기며 끝까지 진행됩니다. quota/오류로 멈추면 다음 실행 때 이어서 수집할 수 있습니다.
4. 수집 시작: 이 단계에서는 NVIDIA AI를 쓰지 않고 유튜브 링크/제목/채널/길이/설명만 저장합니다.
5. 영상 하나를 직접 넣고 싶으면 **유튜브 URL 직접 저장**에 링크를 붙여넣습니다.
6. **2. NVIDIA AI 분석**에서 미분석 항목을 일괄 분석합니다. 이때 메들리 여부와 포함 곡 수가 저장됩니다.
7. **3. 곡 검수**에서 키/BPM/첫 코드/마지막 코드/주제/메들리 포함 곡 수를 확인합니다.
8. 검수 완료 저장
9. **4. 콘티 추천**에서 구간별 빠르기, 곡 수, 이어부르기 여부, 분위기/주제 선택
10. **추천 세트 개수**를 정합니다. 기본값은 7세트이며, 최대 20세트까지 추천받을 수 있습니다.
11. 추천 결과에서 1안/2안/3안처럼 여러 콘티 세트를 비교하고, 최종으로 확인할 세트를 선택합니다.
12. 선택한 콘티의 실제 곡 수 기준 순서와 유튜브 링크를 확인하고, 복사용 콘티를 복사합니다.


## 전체 수집 방식

이 버전은 “검색어당 10개/20개만 수집” 같은 제한을 두지 않습니다.

- YouTube 검색 1페이지는 50개로 요청합니다.
- 응답에 `nextPageToken`이 있으면 다음 페이지를 계속 요청합니다.
- 더 이상 `nextPageToken`이 없으면 해당 검색어는 완료 처리됩니다.
- Google/YouTube quota 초과, 네트워크 오류, API 오류가 나면 현재 페이지 토큰을 저장하고 멈춥니다.
- 다음에 다시 실행하면 **이전 중단 지점부터 이어서 수집**할 수 있습니다.
- 중복 링크/곡은 DB의 `video_id` 기준으로 건너뜁니다.
- 찬양팀 우선순위는 사용하지 않습니다. `worship_sources.csv`에는 `team`, `search_queries`, `notes`만 사용합니다.

기본은 모든 영상을 저장하는 방식입니다. 커버/MR/너무 짧거나 긴 영상 링크를 제외하고 싶을 때만 **제외 키워드/길이 필터 적용**을 켜세요.

## 수집 진행상태 확인

왼쪽 메뉴 **5. DB 관리**에서 검색어별 진행상태를 볼 수 있습니다.

- `completed = 1`: 해당 검색어는 끝까지 수집 완료
- `completed = 0`: quota 초과/오류/중단으로 이어서 수집 가능
- `next_page_token`: 다음 실행 때 이어갈 위치
- `pages_fetched`: 지금까지 가져온 검색 페이지 수
- `videos_seen`: 지금까지 확인한 영상 수
- `songs_inserted`: 신규 저장된 곡 수
- `songs_skipped`: 중복/제외된 곡 수


## 여러 콘티 세트 추천 방식

콘티 추천 화면에서는 한 세트만 보여주지 않고, 조건에 맞는 여러 세트를 점수순으로 보여줍니다.

- `추천 세트 개수`에서 1~20개까지 원하는 개수를 정할 수 있습니다.
- 기본값은 7세트입니다.
- 각 세트는 전체 점수, 실제 곡 수, 앞부분 곡명 미리보기와 함께 표시됩니다.
- `최종으로 확인할 콘티 세트`에서 마음에 드는 세트를 고릅니다.
- 선택한 세트는 표, 유튜브 링크 목록, 복사용 텍스트로 확인할 수 있습니다.
- `전체 추천 세트도 아래에 모두 펼쳐서 보기`를 켜면 선택하지 않은 후보도 함께 비교할 수 있습니다.

NVIDIA AI 콘티 설명은 여러 세트 전체가 아니라, 사용자가 선택한 세트 1개에 대해서만 생성됩니다. 이렇게 해야 AI 요청이 너무 많아져서 429 오류가 나는 것을 줄일 수 있습니다.

## 여러 곡이 한 영상에 들어있는 경우

예를 들어 유튜브 제목이 아래처럼 되어 있으면:

```text
생명 주께 있네+다와서 찬양해+주 우리 아버지+왕의 왕 주의 주+그리 아니하실지라도 - 피아워십
```

앱은 이 영상을 곡별 행으로 쪼개지 않고, DB에는 **메들리 항목 1개**로 저장합니다.

```text
clean_title = 생명 주께 있네 + 다와서 찬양해 + 주 우리 아버지 + ...
is_medley = 1
medley_song_count = 5
medley_songs = ["생명 주께 있네", "다와서 찬양해", "주 우리 아버지", "왕의 왕 주의 주", "그리 아니하실지라도"]
video_url = 원본 유튜브 링크
```

단독곡 영상과 메들리 안에 포함된 곡이 겹쳐도 중복 제거하지 않고 각각 저장합니다.
예를 들어 `생명 주께 있네` 단독 영상도 있고, 위 5곡 메들리 안에도 있으면 둘 다 DB에 남습니다.

콘티 추천에서는 메들리를 1곡으로 세지 않습니다. `medley_song_count` 기준으로 실제 곡 수만큼 카운트합니다.

예시:

```text
요청: 빠른곡 3곡
후보: 빠른곡 메들리 5곡짜리
결과: 그 메들리 전체를 5곡으로 넣지 않고, 필요한 3곡만 사용하는 추천 가능
```

추천 결과에는 `순서 4~6`, `곡 수 3`, `전체 메들리 5곡 중 3곡만 사용`처럼 표시됩니다.

## 예시 콘티 조건

- 구간 1: 느린곡 3곡 / 이어부르기 / 은혜, 임재, 회복, 경배
- 구간 2: 빠른곡 3곡 / 이어부르기 / 찬양, 기쁨, 감사, 선포, 승리
- 구간 3: 느린곡 1곡 / 끊어서 / 결단, 기도, 임재, 회복

## 주의사항

- 이 프로그램은 악보나 가사를 무단으로 크롤링하거나 저장하지 않습니다.
- YouTube 검색 결과와 AI 추정 결과는 완벽하지 않으므로 반드시 검수 단계를 거쳐야 합니다.
- “제한 없이 전체 수집”으로 설정했더라도 YouTube Data API의 일일 quota와 YouTube가 검색 API로 제공하는 페이지 범위는 넘길 수 없습니다. quota가 끝나면 다음 실행 때 이어서 수집하세요.
- NVIDIA API는 곡명/분위기/메들리 포함 곡 수 분석 보조용입니다. 키, BPM, 첫 코드, 마지막 코드는 실제 예배팀 기준으로 검수하는 것을 권장합니다.
